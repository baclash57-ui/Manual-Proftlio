## Test Plan

### Test Plan Objective

The objective of this test plan is to validate the core functionalities and user experience of the Books-A-Million e-commerce website. The testing focuses on ensuring that users can successfully browse, search for, select, and purchase books with accurate system behavior, proper navigation, and consistent pricing.

---

### Test Scope

Testing will cover the following high-level areas:

- Homepage functionality and layout
- Search and filter behavior
- Product description accuracy
- Cart operations and price calculations
- General usability and link responsiveness

Features not directly related to the purchasing flow or core browsing experience are excluded from this test plan.

---

### Test Approach

A **manual functional testing** approach will be used to evaluate the application from an end-user perspective. Testing will focus on:

- Core user flows and critical paths
- Validation of expected behaviors for valid and invalid actions
- Visual and usability checks for layout and responsiveness
- Navigation and link redirection accuracy

Test scenarios and test cases will be derived from the defined in-scope areas and executed across supported pages of the website.

---

### Test Environment

Testing will be performed in a web browser environment using a desktop device and microsoft Edge browser.

The focus will be on verifying functionality in a stable test environment that mirrors real user interaction conditions.

---

### Entry Criteria

- Application is accessible and loads without critical errors
- Core pages such as homepage, product pages, and cart are available
- Test data (books, prices, categories) is present

---

### Exit Criteria

- 90% planned test cases have been executed
- Critical and high-severity defects have been identified and documented
- Core user flows function as expected without blocking issues

---

### Deliverables

- Test scenarios
- Test cases
- Defect reports (where applicable)
