**BG_LN_001**.

**Title:** Several sections in the Attractions page show content in different languages after the user changes language

**Type:** Functional

**Severity:** Medium

**Environment:** Windows 11

**Browser:** Microsoft Edge

**Summary:** 

After changing the site language, several sections on the Attractions page continue to display content in other languages. The language change is not applied consistently across all sections of the Attractions page.

**Steps to Reproduce:**

1. Open Booking.com
2. Navigate to the Attractions section from the navigation header
3. Locate the language selector in the header
4. Select a preferred language
5. Observe the content displayed across the Attractions page

**Expected Result:**

 All content on the Attractions page updates to display in the selected language after the change is made

**Actual Result:**

 Several sections on the Attractions page continue to display content in other languages despite the language change being applied

**Bug ID:** BG_CR_001

**Title:** Some sections of the Attractions destination listing page do not display prices in the user's local currency

**Type:** Functional

**Severity:** Medium

**Environment:** Windows 11

**Browser:** Microsoft Edge

**Summary:** On the Attractions destination listing page, several sections do not display prices in the user's local currency. The prices in these sections appear in a different currency despite the user's local currency being set on the site.

**Precondition:** The user should be using the site with their local currency.

**Steps to Reproduce:**

1. Open Booking.com
2. Navigate to the Attractions section from the navigation header
3. Scroll down to the Explore More Destinations section
4. Click on any destination card to open the destination listing page
5. Observe the prices displayed across the different sections of the page

**Expected Result:** All sections on the destination listing page display prices in the user's local currency

**Actual Result:** Several sections on the destination listing page do not display prices in the user's local currency

**Bug ID:** BG_CR_002

**Title:** User cannot change their currency back after switching it on the site

**Type:** Functional

**Severity:** Medium

**Environment:** Windows 11

**Browser:** Microsoft Edge

**Summary:** 

After switching the currency on Booking.com, the user is unable to change the currency back to their original or any other preferred currency. The currency selector does not display the user’s local currency after a currency change has been made.

**Precondition:** User’s local currency is not internationally traded.

**Steps to Reproduce:**

1. Open Booking.com
2. Change the currency of the site 
3. Attempt to change the currency back 

**Expected Result:** 

The user is able to change the currency freely at any point on the site and return to their local currency

**Actual Result:** The user is unable to change the currency back to their local currency after switching it
